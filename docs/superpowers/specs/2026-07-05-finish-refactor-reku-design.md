# whisper_ptt → Reku: финишный рефакторинг (мердж Фазы 1, структура, установка одной командой)

**Дата:** 2026-07-05
**Статус:** дизайн утверждён в брейншторме, ждёт ревью спецификации перед планом
**Предыдущие спеки:** `2026-06-04-portable-exe-cross-platform-design.md` (Фаза 1),
`2026-07-04-phase2-openvino-igpu-design.md` (Фаза 2)

## Цель

Довести whisper_ptt до состояния «полноценный продукт для себя и друзей»:

1. вернуть в main неслитую июньскую работу (темы, галочки, exe-сборка, надёжность,
   фикс латиницы) и совместить её с влитой Фазой 2 (OpenVINO iGPU);
2. переименовать продукт в **Reku** (заголовок окна, трей, сборка, репозиторий);
3. навести порядок в структуре репозитория (25+ файлов в корне → пакет + подпапки);
4. сделать установку одной командой PowerShell (скрипт сам определяет железо);
5. довести UI: видимость выбранного железа, оценка палитры тёмной темы;
6. подготовить репозиторий к публикации (README, LICENSE) — сама публикация
   выполняется отдельно, только по явной команде пользователя.

## Контекст и текущее состояние

- **main** (13 коммитов после точки расхождения `960cb16`): Фаза 2 — OpenVINO
  GenAI-бэкенд для Intel iGPU/NPU, авто-цепочка device `auto → cuda → igpu → cpu`,
  OV-модели с HF, бенч, авто-фолбэк на CPU при сбое OV. Влита через PR #2.
- **origin/feature/desktop-reliability-ui-installer** (6 коммитов от той же точки,
  НЕ влита): надёжность ядра + фикс латиницы в русских словах; светлая/системная
  тема + состояние «Ошибка» + многострочный словарь + иконка + автозапуск;
  сборка .exe (PyInstaller `--onedir`) + инсталлятор (Inno Setup, per-user) +
  `build.ps1`; фиксы по ревью PR #1 (читаемые цвета, галочка/точка в индикаторах
  чекбоксов/радио); удаление legacy `tray.py`; тесты `test_robustness.py`,
  `test_theme.py`, `test_frozen_smoke.py`, `ab_test.py`.
- Репетиция мерджа (`git merge-tree`): конфликты только в `gui.py`, `dictate.py`,
  `model_store.py`; остальные 7 общих файлов сливаются автоматически.
- Репозиторий приватный (`Small-coder-AI/whisper_ptt`). В истории git нет личных
  данных (проверено: аудио, config.json, логи, ключи не коммитились).
- Окружение ноутбука: Python 3.12.10, venv 1,2 ГБ (Intel-стек без nvidia-пакетов).
- Известная проблема с прошлых сессий: латиница/разрывы слов на длинных диктовках
  (RTX-ПК). Фикс предположительно содержится в ветке Фазы 1 — проверяется после
  мерджа.

## Решения брейншторма

- **Имя продукта: Reku** (от «реку» — др.-рус. «говорю»). Масштаб продукта —
  «для себя и друзей», поэтому коллизия имени с индонезийской криптобиржей Reku
  и занятость `reku` на PyPI/npm признаны несущественными: распространение —
  прямой ссылкой/командой, публикация на PyPI/npm не планируется. Имя репозитория
  GitHub `Small-coder-AI/reku` свободно. Если продукт когда-либо пойдёт в широкую
  публику — имя пересматривается.
- **Порядок работ: мердж → структура → установка.** Перемещение файлов до мерджа
  недопустимо: конфликты «переименован + изменён» многократно усложняют слияние.
- **Способ установки: PowerShell one-liner** (bootstrapper). Классический
  exe-инсталлятор (Inno Setup) сохраняется как запасной офлайн-вариант.
- **Публикация репозитория** — последний шаг, только после вычистки и по явной
  команде пользователя. До неё установка отлаживается локально на ноутбуке.

## Архитектура работ

### Этап 1. Мердж ветки Фазы 1 в main

- Рабочая ветка `merge/phase1-desktop` от main; `git merge
  origin/feature/desktop-reliability-ui-installer`.
- Разрешение конфликтов, принципы совмещения:
  - `gui.py` — из ветки: менеджер тем (светлая/тёмная/системная), состояние
    «Ошибка», словарь, автозапуск-чекбокс, иконка; из main: пункты устройств
    Intel GPU/NPU, модель large-v3-turbo, вывод ошибок движка, откат
    model/device/compute при неудачном reload. Обе группы правок сохраняются.
  - `dictate.py` — из ветки: правки надёжности ядра и фикс латиницы; из main:
    маршрутизация бэкендов и авто-фолбэк OpenVINO → CPU. Обе сохраняются.
  - `model_store.py` — из ветки: прогресс/устойчивость скачивания; из main:
    OV-модели (snapshot_download, маркер завершённости). Обе сохраняются.
  - `tray.py` — удалён (решение ветки, main его уже не использует).
- `gui_theme.py`: в ветке файл переписан под несколько тем (в main тем временем
  добавлено состояние `downloading` — см. коммит `03db363`). Проверить, что все
  состояния (`loading`, `downloading`, `idle`, `recording`, `transcribing`,
  `error`) имеют цвет и подпись во всех темах.
- Критерий готовности: все тесты обеих веток проходят
  (`test_postprocess`, `test_backends`, `test_paths`, `test_transcribe_pipeline`,
  `test_robustness`, `test_theme`, `test_frozen_smoke`, `smoke_gui`,
  `selftest_pipeline`); ручная проверка на ноутбуке — темы переключаются,
  индикаторы чекбоксов/радио отображаются корректно, диктовка на iGPU работает,
  латиница на длинной диктовке не появляется (закрывает памятку
  halluc-long-texts при подтверждении).
- Мердж в main через PR (как в Фазе 2).

### Этап 2. Переименование в Reku + структура репозитория

Целевая раскладка (корень: 6–7 файлов вместо 25+):

```
reku/                        # репозиторий (после rename)
  reku/                      # пакет приложения
    __init__.py, __main__.py # запуск: pythonw -m reku
    gui.py, gui_theme.py, gui_widgets.py
    dictate.py, backends.py, postprocess.py
    config.py, model_store.py, cuda_setup.py, autostart.py
  tests/                     # test_*.py, smoke_gui.py, selftest_pipeline.py
  scripts/                   # diag.py, diag_halluc.py, diag_paste.py,
                             # bench_backends.py, ab_test.py, render_preview.py,
                             # make_ico.py (генерирует packaging/app.ico)
  packaging/                 # reku.spec, reku.iss, build.ps1, app.ico
  docs/                      # без изменений
  install.ps1                # установка одной командой (этап 3)
  README.md, LICENSE, requirements.txt, requirements.lock.txt, .gitignore
```

- Импорты внутри пакета — абсолютные (`from reku import config`); тесты и
  скрипты обновляются; `cuda_setup` по-прежнему импортируется до
  `faster_whisper` (порядок сохраняется).
- `smoke_gui.py` и `selftest_pipeline.py` переезжают в `tests/`; бенчи и
  диагностика — в `scripts/`; PyInstaller spec и Inno Setup — в `packaging/`
  с правкой путей внутри.
- Переименование продукта:
  - заголовок окна, тултип трея, single-instance сокет — «Reku»;
  - PyInstaller: `name='Reku'` → `dist/Reku/Reku.exe`; Inno Setup:
    `MyAppName Reku`, установка в `%LOCALAPPDATA%\Programs\Reku`;
  - `data_dir()` (frozen): `%APPDATA%\Reku`; **мягкая миграция** — если
    `%APPDATA%\whisper_ptt` существует, а `%APPDATA%\Reku` нет, каталог
    переименовывается при старте (модели ~1,5–3 ГБ не перекачиваются;
    при запуске из исходников data_dir остаётся каталогом проекта);
  - README и docs — новое имя;
  - репозиторий GitHub переименовывается в `reku` (старый URL автоматически
    перенаправляется), локальный remote обновляется.
- Критерий готовности: приложение запускается `pythonw -m reku`, все тесты
  проходят из новой раскладки, `git log --follow` сохраняет историю файлов
  (перемещения — отдельным коммитом от содержательных правок).

### Этап 3. Установка одной командой (`install.ps1`)

Сценарий пользователя: `irm https://raw.githubusercontent.com/Small-coder-AI/reku/main/install.ps1 | iex`
(пока репозиторий приватный, отладка — локальным запуском файла).

Скрипт (идемпотентный; повторный запуск = обновление):

1. **Детект железа:** NVIDIA — через `nvidia-smi` (есть в PATH при драйвере)
   или WMI `Win32_VideoController`; Intel GPU — по имени адаптера в WMI.
   Итог: профиль `cuda` / `intel` / `cpu`.
2. **Python 3.12:** если подходящий Python (3.12.x) не найден — установка через
   `winget install Python.Python.3.12` (тихий режим); фолбэк при отсутствии
   winget — официальный установщик python.org с ключами тихой установки.
3. **Код:** скачивание zip-архива ветки main с GitHub (`Invoke-WebRequest`,
   git не требуется) в `%LOCALAPPDATA%\Programs\Reku-src` (или обновление
   существующего: старый каталог заменяется новым с сохранением venv).
4. **Окружение:** `python -m venv .venv`; установка зависимостей по профилю:
   `cuda` — полный `requirements.txt`; `intel`/`cpu` — без `nvidia-*` пакетов
   (фильтрация строк). Профиль `intel` включает `openvino-genai` (уже в
   requirements). Прогресс — стандартный вывод pip.
5. **Ярлыки:** Пуск и (опционально, вопросом) рабочий стол — на
   `pythonw.exe -m reku` с иконкой `packaging/app.ico`; вопрос про автозапуск
   (HKCU `...\Run`, тот же механизм, что в `autostart.py`).
6. **Финал:** сообщение «Готово, запускай Reku из Пуска; модель скачается при
   первом запуске».

Дополнительно: `install.ps1 -Uninstall` — удаление ярлыков, автозапуска,
каталога программы; данные (`%APPDATA%\Reku`, модели) — с отдельным вопросом.

Критерий готовности: чистая установка на ноутбуке (текущая ручная установка
удаляется, ставится скриптом), диктовка работает на iGPU; повторный запуск
скрипта не ломает установку.

### Этап 4. Доводка UI

- В настройках — строка фактического рантайма: «Работает: Intel GPU (OpenVINO) ·
  модель large-v3» (данные `device_label` бэкенда уже прокинуты в подсказку —
  выводится на видное место).
- Оценка палитры после мерджа (в ветке она уже переработана на читаемость):
  если тёмная тема всё ещё воспринимается «слишком чёрной» — фон поднимается
  на 1–2 тона (правки только констант `gui_theme.py`).
- Ручная проверка обеих тем на ноутбуке.

### Этап 5. Предпубликационная уборка (публикация — отдельно, по команде)

- README: что за продукт, скриншот, команда установки, требования, FAQ
  (кратко, по-русски; английская версия — опционально позже).
- LICENSE: MIT.
- Вычистка: `bench_run.log` и прочие артефакты из рабочей копии (в git они
  не попадали), актуализация `.gitignore` под новую раскладку.
- Файл `whisper_ptt.iss`/`spec` — уже переименованы на этапе 2; проверка, что
  `build.ps1` собирает `Reku.exe` (запасной офлайн-путь распространения).
- **Не в этом этапе:** само переключение репозитория в public — выполняется
  отдельной командой пользователя после его финального осмотра.

## Обработка ошибок

- Мердж: при неоднозначном конфликте приоритет — сохранить поведение обеих
  фаз; спорные места фиксируются в PR-описании для ревью.
- Миграция data_dir: при ошибке переименования (каталог занят другим
  процессом) — работа продолжается со старым путём, в лог пишется
  предупреждение; повторная попытка при следующем старте.
- install.ps1: каждый шаг с понятным сообщением об ошибке (нет интернета,
  нет winget, нет прав); скрипт останавливается на первом сбое
  (`$ErrorActionPreference = "Stop"`), не оставляя полуустановки без сообщения.
- Детект железа в скрипте консервативен: сомнение → профиль `cpu` (приложение
  само поднимет iGPU через device=auto, если он есть).

## Тестирование

- Все существующие тесты обеих веток — после каждого этапа.
- Новые: `test_paths`-подобная проверка миграции `%APPDATA%\whisper_ptt` →
  `%APPDATA%\Reku` (мок файловой системы/temp-каталоги); smoke импорта пакета
  `reku` из новой раскладки.
- `install.ps1` — ручная проверка на ноутбуке: чистая установка, повторный
  запуск (обновление), `-Uninstall`.
- Ручная проверка UI на ноутбуке: темы, чекбоксы/радио, диктовка iGPU,
  длинная диктовка (латиница).

## Риски

- **Мердж-конфликты сложнее ожидаемого** (gui.py: темы × OV-устройства) —
  смягчение: репетиция показала локальность конфликтов; тесты обеих веток
  как ворота.
- **Перемещение файлов ломает PyInstaller spec / frozen-пути** — смягчение:
  `test_frozen_smoke.py` из ветки + контрольная сборка `build.ps1` на этапе 5.
- **install.ps1 на чужой машине** (нет winget, корпоративные политики,
  антивирус на `irm | iex`) — смягчение: понятные сообщения об ошибках,
  запасной путь — exe-инсталлятор.
- **Миграция каталога данных** на ПК с RTX (большие модели) — смягчение:
  переименование каталога атомарно в пределах диска; фолбэк на старый путь.

## Вне скоупа

API-бэкенд (заглушка остаётся), история диктовок, плавающий индикатор у
курсора, автообновления, английская локализация UI, публикация на PyPI,
CI/CD. Публикация репозитория — вне скоупа этапов 1–5, отдельное действие
по команде пользователя.
