# whisper_ptt → переносимый .exe + кросс-платформенный инференс

**Дата:** 2026-06-04
**Статус:** дизайн утверждён, ждёт ревью спецификации перед планом
**Фаза этого документа:** Фаза 1 (на текущем ПК с RTX). Фазы 2–3 — в разделе «Дорожная карта».

## Цель

Превратить whisper_ptt из «запусти команду в venv» в полноценное приложение для Windows:
двойной клик → работает, автозапуск в трей, авто-детект железа (CUDA → GPU, иначе → CPU),
не падает на не-NVIDIA машинах, корректно хранит настройки, не запускается дважды,
плюс несколько удобств из аналогов. Архитектуру сразу делаем с абстракцией бэкенда,
чтобы Intel/OpenVINO встал в Фазе 2 без переписывания.

## Контекст и текущее состояние

Сейчас работает: ядро `DictationApp` (запись → распознавание → вставка) чисто отделено
от UI на PySide6 (`gui.py`), есть фильтры галлюцинаций (`postprocess.py`), CUDA-обвязка
(`cuda_setup.py`), тесты постпроцессинга. GPU-инференс честный (~0.7 с на короткую фразу,
RTX 3050, float16, large-v3).

Боли, которые закрываем:
- Запуск только командой из venv — нет ярлыка/автозапуска, забыл команду → не запустил.
- Сборки .exe нет вообще (ни спека, ни билда).
- `device="cuda"` зашит намертво — на не-NVIDIA краш без понятного сообщения.
- `config.json` лежит рядом со скриптом — во frozen-сборке в `Program Files` это read-only.
- Нет защиты от второго инстанса (README сам предупреждает «держи один») → двойная VRAM.
- Нет first-run UX (прогресс скачивания модели, сообщение про CPU-фолбэк).

## Решения, принятые на брейншторме

- **Аудитория:** «мои машины + изредка дать другим» → **одна универсальная сборка** с
  авто-детектом железа; модель **скачивается при первом запуске** (не вшита).
- **Intel-ноут (Honor MagicBook Art 14, Core Ultra 7 255H, Arc 140T + NPU):** CTranslate2
  не умеет Intel-GPU. Сейчас на Intel работаем на **CPU**; **OpenVINO (iGPU/NPU) —
  Фаза 2**, реализуем на самом ноуте (стянув код с GitHub), где удобнее тестировать.
  В Фазе 1 делаем только **задел** — абстракцию бэкенда.
- **Скоуп Фазы 1:** все 4 блока (запуск без команд, авто-детект, задел+надёжность, фичи).
- **Упаковка:** PyInstaller **`--onedir`** (не `--onefile` — тот распаковывает ~ГБ во
  временную папку на каждом старте).

Факты подтверждены через context7:
- `WhisperModel(model, device="cuda"|"cpu", compute_type=...)` — авто-режима device нет,
  детектим сами. Модели качаются с HF Hub; `download_model(name, output_dir=, local_files_only=)`
  даёт контроль места, проверку «уже скачано» и точку для прогресса. `available_models()`
  отдаёт список имён.
- PyInstaller: `collect_all(pkg) -> (datas, binaries, hiddenimports)`, `collect_dynamic_libs`
  для DLL. PySide6 покрыт встроенным хуком.

## Архитектура Фазы 1

### A. Авто-детект железа + абстракция бэкенда

Новый модуль `backends.py`:

- **`Backend`** (ABC/Protocol) — интерфейс:
  - `load()` — загрузить модель;
  - `transcribe(audio: np.ndarray, opts) -> (segments, info)` — сырой вызов движка;
  - свойства `name` (например `"ctranslate2"`) и `device_label` (например `"GPU (CUDA)"`/`"CPU"`).
- **`CTranslate2Backend`** — единственная реализация сейчас; в неё переезжает текущая
  логика `load_model()` и вызова `self.model.transcribe(...)` из `dictate.py`.
- **`OpenVINOBackend`** — **только заглушка интерфейса** (класс с `NotImplementedError`),
  чтобы зафиксировать контракт. Реализация — Фаза 2.
- **`detect_device() -> (device, compute_type)`**:
  - если `ctranslate2.get_cuda_device_count() > 0` → `("cuda", "float16")`;
  - иначе → `("cpu", "int8")`.
  - (Если API `get_cuda_device_count` окажется недоступен в нашей версии ct2 — фолбэк:
    попытка создать крошечную модель на cuda в try/except. Уточняется на этапе плана.)
- **Авто-выбор модели по устройству:** при `device="auto"` и реальном CPU дефолтная
  тяжёлая `large-v3` заменяется на **`small` int8** (приоритет — отзывчивость PTT;
  large-v3 на CPU слишком медленный). Пользователь может поднять до `medium` в настройках.
  Финальный CPU-дефолт перепроверим на реальном ноуте в Фазе 2. Выбор пишется в конфиг.

Изменения в `dictate.py`: `DictationApp` хранит `self.backend` вместо прямого `self.model`;
`load_model()`/`reload_model()` делегируют бэкенду; `transcribe()` зовёт `self.backend.transcribe(...)`,
а фильтры/вставка остаются как есть.

Изменения в `config.py`: поле `device` принимает `"auto"` (значения: `auto`/`cuda`/`cpu`);
`compute_type` тоже может быть `"auto"` (выводится из device). Дефолт после миграции — `"auto"`.

Изменения в UI (`gui.py`): в комбобоксе «Устройство» добавляется пункт «Авто»; статус
показывает реально выбранное устройство (`device_label` бэкенда); при фолбэке на CPU —
разовое уведомление «GPU не найден → работаю на CPU».

### B. Хранение конфига, данных и защита от второго инстанса

- **`data_dir()`** в `config.py`: если `getattr(sys, "frozen", False)` — `%APPDATA%\whisper_ptt\`;
  иначе (из исходников) — каталог скрипта (как сейчас, для удобства разработки).
- `config.json`, лог и кэш моделей живут в `data_dir()`. Кэш моделей →
  `%APPDATA%\whisper_ptt\models` (стабильно, легко найти/удалить); путь передаётся в
  faster-whisper через `download_root`/`HF_HOME`.
- **Защита от второго инстанса:** `QLocalServer`/`QLocalSocket` (мы уже в Qt). Первый
  инстанс слушает именованный сокет; второй — коннектится, шлёт «покажись», и выходит.
  Первый по сигналу показывает окно из трея.

### C. Упаковка и запуск без команд

- **PyInstaller `--onedir --windowed`**, точка входа `gui.py`, иконка `.ico`
  (сгенерируем из существующего тёмного orb'а в `gui.py:make_icon`).
- **`whisper_ptt.spec`**:
  - `collect_all("ctranslate2")`, `collect_all("faster_whisper")`;
  - сбор nvidia-DLL: `collect_dynamic_libs("nvidia.cublas")`, `"nvidia.cudnn"`,
    `"nvidia.cuda_nvrtc"` (имена пакетов уточним на этапе плана);
  - PySide6 — встроенным хуком; неиспользуемые Qt-модули исключаем (`excludes`) для размера;
  - hidden imports под VAD-зависимости faster-whisper (`onnxruntime`/`av`), если используются.
- **`cuda_setup.py` — frozen-aware:** ищет nvidia-DLL и относительно `sys._MEIPASS`/каталога
  .exe (frozen), и в `site-packages` (из исходников). **Главный риск упаковки** — лечим
  здесь и проверяем смоук-тестом собранного .exe.
- **Модель не вшиваем.** Первый запуск скачивает модель → **состояние «скачиваю модель…»**
  в UI с прогрессом (через колбэк huggingface_hub или индикатор «N МБ»); проверка
  «уже скачано» через `download_model(..., local_files_only=True)`.
- **Универсальность:** на не-NVIDIA CUDA-DLL просто лежат балластом — ct2 уходит на CPU.
  Тот же .exe заведётся на ноуте на CPU; OpenVINO добавится в Фазе 2. (Опционально позже —
  отдельная «лёгкая» CPU-сборка без CUDA-DLL для меньшего размера.)
- **Автозапуск:** чекбокс «Запускать при старте Windows» в настройках → пишет/удаляет
  значение в `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`.
- **`build.ps1`:** запускает PyInstaller, создаёт ярлык на рабочем столе/в Пуске
  (через WScript.Shell COM), опционально зипует `dist\whisper_ptt\` для передачи.
- Результат: `dist\whisper_ptt\whisper_ptt.exe` — двойной клик, иконка в трее,
  опционально автозапуск. Команды venv больше не нужны.

### D. Фичи из аналогов (по-минимуму)

- **Звук старт/стоп:** короткие сигналы через `winsound`; тумблер `sound_feedback` в конфиге.
- **История диктовок:** последние N результатов в памяти + persist в `history.jsonl`
  (в `data_dir()`); отдельная страница/пункт меню; клик по записи = скопировать в буфер.
- **Поле «Словарь терминов»:** многострочный ввод в настройках, привязанный к существующему
  `initial_prompt` (логика уже есть в `transcribe`).
- В Фазу 3: плавающий индикатор у курсора, локальное AI-форматирование.

## Поток данных (без изменений по сути)

Хоткей → `start_rec()` пишет аудио → `stop_and_transcribe()` → `backend.transcribe()` →
фильтры `postprocess` → `insert()` (буфер + Ctrl+V) → `on_result` в UI + запись в историю.
Новое: при старте `detect_device()` выбирает бэкенд/устройство; при первом запуске —
скачивание модели с прогрессом.

## Обработка ошибок

- Нет CUDA/не-NVIDIA → авто-фолбэк на CPU + уведомление в UI (не краш).
- Сбой загрузки модели (нет сети при первом скачивании, OOM) → понятное сообщение в UI,
  приложение остаётся живым (можно сменить модель/устройство в настройках и повторить).
- Второй инстанс → показать существующее окно, выйти (без второй модели в памяти).
- Конфиг нечитаем/каталог недоступен → дефолты + лог, не падаем.

## Тестирование

- Существующие тесты (`test_postprocess.py`, `selftest_pipeline.py`) сохраняем.
- Добавляем:
  - `detect_device()` — юнит-тест с моком ct2 (есть CUDA / нет CUDA).
  - `config.data_dir()` — frozen vs из исходников.
  - контракт `Backend` — `CTranslate2Backend` реализует интерфейс; `OpenVINOBackend`
    кидает `NotImplementedError`.
  - защита от второго инстанса — мок QLocalServer/смоук.
- **Ручная проверка собранного .exe** (нужен реальный запуск): двойной клик → трей →
  загрузка модели → диктовка → вставка; автозапуск; second-instance.

## Риски

- **CUDA-DLL во frozen-сборке (риск №1):** ct2 грузит их через голый LoadLibrary; лечим
  frozen-aware `cuda_setup.py` + обязательный смоук-тест собранного .exe.
- **Размер onedir-сборки** ~2–3 ГБ до модели (CUDA-DLL). Приемлемо для выбранной аудитории.
- **Intel/OpenVINO здесь не тестируется** (нет железа) — осознанно перенесено в Фазу 2.

## Дорожная карта

- **Фаза 1 (сейчас, RTX-ПК):** этот документ — запуск без команд, авто-детект, задел+надёжность, фичи.
- **Фаза 2 (на ноуте Honor):** реализация `OpenVINOBackend` на iGPU/NPU на реальном железе;
  возможно отдельная CPU-only «лёгкая» сборка.
- **Фаза 3 (полировка):** плавающий индикатор у курсора, локальное AI-форматирование
  (локальная LLM или опционально Claude API), доп. режимы записи (стоп по тишине).

## Вне скоупа Фазы 1

Облако/аккаунты/подписки (убивают приватность — главное преимущество), OpenVINO-реализация,
AI-форматирование, плавающий оверлей, инсталлятор для незнакомых пользователей.
